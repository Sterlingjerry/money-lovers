==================================================
FILE OWNERSHIP (PRIMARY)
==================================================
Frontend
1) Jose
   Owns:
   - frontend/src/index.jsx
   - frontend/src/styles.css
   - frontend/src/index.js
   - frontend/public/index.html
   - frontend/package.json (frontend dependency updates only)
   Duties:
   - Build and maintain login/register/dashboard UI
   - Handle user interactions and page flow
   - Keep layout responsive and beginner-friendly
   - Coordinate with Simi for API payload changes

2) Jose + Simi (shared)
   Owns:
   - frontend/src/api.js
   Duties:
   - Jose: frontend request usage and error handling UX
   - Simi: endpoint path consistency and API contract alignment

Backend
3) Simi
   Owns:
   - backend/routes.py
   - backend/app_factory.py
   - backend/run_simple.py
   - backend/requirements.txt (backend dependency updates only)
   Duties:
   - Create and maintain API endpoints
   - Validate request input and return correct status codes
   - Maintain business flow for subscriptions and payments
   - Keep API behavior aligned with frontend needs

4) Riad
   Owns:
   - backend/models.py
   Duties:
   - Maintain User, Subscription, Payment schema
   - Manage relationships and data rules
   - Review model-level changes for integrity and consistency
   - Coordinate with Simi when route logic depends on schema updates

5) Sterling
   Owns (security review authority):
   - backend/models.py (auth/security sections)
   - backend/routes.py (auth/access checks)
   - frontend/src/api.js (token handling checks)
   Duties:
   - Ensure bcrypt password hashing remains intact
   - Verify account lockout behavior after failed logins
   - Verify route protection and ownership-based access control
   - Review auth/token flow for regressions

==================================================
CHANGE RULES
==================================================
Rule 1
- If you modify a file owned by someone else, tag them for review.

Rule 2
- Schema change in backend/models.py requires:
  - Riad approval
  - Simi confirmation that routes still work
  - Jose confirmation if frontend payloads change

Rule 3
- Auth/security change requires Sterling review before merge.

Rule 4
- API endpoint shape change requires:
  - Simi updates backend/routes.py
  - Jose updates frontend/src/index.jsx usage

Rule 5
- Keep comments clear and simple for class presentation purposes.
